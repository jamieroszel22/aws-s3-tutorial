Create a static website with Amazon S3

In this tutorial, we will build and host a static webpage on Amazon S3.

What is Amazon S3 and How Do Static Sites Work?

Amazon S3 (Simple Storage Service) is a cloud-based object storage service from Amazon Web Services (AWS). It's designed to store and retrieve virtually unlimited amounts of data from anywhere on the web. Think of it as a super-powered online hard drive for your files.

For more information on S3, check out the video below:

Static websites are built entirely with HTML, CSS, and maybe some client-side JavaScript. They don't rely on a server to generate content on the fly, making them simple, fast, and often more affordable to host.

Getting Started

  1. Sign up for AWS Free Tier: Go to the AWS free tier page and sign up for an account.
  2. Create an S3 Bucket:
    • Open the AWS S3 console and create a new bucket. Give it a unique name.
    • Once your bucket is created, navigate to the Properties tab.
    • Next, navigate to Static website hosting and click Edit.
    • Select "Enable" under Static website hosting.
    • Set the index document to index.html.
  3. Enable Public Access to an S3 Bucket:
    • Open the Permissions tab: Click the "Permissions" tab within your bucket's settings.
    • Locate the Block Public Access section: Scroll down to find the section titled "Block public access (bucket settings)".
    • Edit the settings: Click the "Edit" button next to the Block Public Access settings.
    • Uncheck Block all public access: Locate the checkbox labeled "Block all public access" and uncheck it.
    • Save Changes: Click the "Save Changes" button to apply the modifications.
    • Confirmation: Type "confirm" into the provided field to verify that you understand the security implications and intentionally wish to make the bucket public.
    • Final Save: Click "Save Changes" once more to finalize the process.
  4. Apply a Bucket Policy:
    • Purpose: A bucket policy defines the access permissions for your S3 bucket. In this case, we'll create a policy to allow public read access for making your website accessible.
    • Navigate to Permissions: In the S3 console, go to your bucket's "Permissions" tab.
    • Bucket Policy Section: Click on the "Bucket Policy" section.
    • Paste the Policy: Paste the following JSON code into the bucket policy editor, replacing 'your-bucket-name' with your actual bucket name: 
      
                    { 
                      "Version": "2012-10-17", 
                      "Statement": [ 
                        { 
                          "Sid": "PublicReadForGetBucketObjects", 
                          "Effect": "Allow", 
                          "Principal": "*", 
                          "Action": "s3:GetObject", 
                          "Resource": "arn:aws:s3:::your-bucket-name/*" 
                        } 
                      ] 
                    }

    • Bucket Policy Explanation

      Overall Purpose: This policy is designed to grant public read access to all objects within a specific S3 bucket. This is generally used when hosting static websites on S3.

      Key Elements:

      • Version: "2012-10-17" Specifies the version of the policy language being used.
      • Statement: [ ... ] This is an array containing one or more individual policy statements. In this case, we have a single statement. Let's break down that statement:
        • Sid: "PublicReadForGetBucketObjects" A "Statement ID", a simple identifier or label for this particular statement.
        • Effect: Allow Determines whether the statement grants permissions (Allow) or denies them (Deny).
        • Principal: "*" Specifies who the policy applies to. The asterisk (*) means "everyone" (i.e., public access).
        • Action: s3:GetObject The specific Amazon S3 action that's being allowed. In this case, it's the ability to read/retrieve objects from the bucket.
        • Resource: "arn:aws:s3:::your-bucket-name/*" The Amazon Resource Name (ARN) specifying the S3 bucket and objects to which this policy applies. You'll need to replace "your-bucket-name" with your actual bucket name. The asterisk at the end (*) means all objects inside the bucket are affected.
    • Save Changes: Click "Save" to apply the bucket policy.
  5. Upload Files: Upload your HTML files (including this index.html file) and any other assets (images, videos, etc.) to the S3 bucket.
  6. Your Website: Access your website using the endpoint provided by Amazon S3. It will look something like: http://your-bucket-name.s3-website.your-region.amazonaws.com

Additional Notes

  • You can use any text editor to create your HTML files.
  • You can upload images, videos, and other assets to your S3 bucket.
  • You can add client-side scripts to your website to make it more interactive.
  • Once you are done with your website, you can delete the S3 bucket to remove it from the internet.
